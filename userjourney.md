# Aura Spatial User Journey

This document outlines the user journey for the Aura Spatial platform.

## Initiation (The Spark)

*   **Client:**
    *   Furnishes information such as specifications of the house.
    *   Explores themes and defines the scope of the project.
    *   Chooses to "Proceed to receive quotations."

*   **System:**
    *   **Task:** Transforms the client's inputs into a professional Request-For-Quote (RFQ).
    *   **Output:** Publishes the RFQ to the marketplace for Interior Designers (IDs) to bid on the project.

## Bidding and Selection

*   **Interior Designer (ID):**
    *   Receives the RFQ and submits a competitive bid.

*   **Client:**
    *   Receives bids from various IDs and makes a selection.

*   **System:**
    *   **Task:** Notifies the selected ID and the client about the match.

## Project Creation

*   **ID and Client:**
    *   Discuss the project scope and determine the tasks in detail.
    *   The system provides a notetaking facility to log all requirements.

*   **ID and System:**
    *   The ID uses the system to build a book-of-work and a detailed project schedule.
    *   The system produces a full Work-Breakdown-Structure (WBS) for the project.

## Execution & Synthesis

*   **Interior Designer (ID):**
    *   Manages the project and provides timely status updates through the platform.

*   **Client:**
    *   While the ID works on the project, the client can experiment with 3D renderings of their space.
    *   The client can start shopping for furniture and decor through the integrated e-commerce marketplace.
